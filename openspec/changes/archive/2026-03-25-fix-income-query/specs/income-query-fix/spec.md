# Income Query Fix Specification

## Summary

修复收入查询页面的数据计算和显示问题，确保占比计算不会出现 NaN，并优化押金支出的显示逻辑。

## Requirements

### 1. 占比计算修复
- 当总收入为 0 时，所有收入类型的占比应显示为 '0%' 而不是 NaN
- 确保除法运算的安全性

### 2. 押金支出处理优化
- 押金支出作为负数金额，在计算占比时应正确处理
- 在显示时明确标识为支出项目

### 3. 数据格式化改进
- 确保所有金额计算的精度和安全性
- 改进数值格式化过程

## API Changes

无 API 变更要求。

## UI Changes

### 收入明细表格
- 修复占比列的 NaN 显示问题
- 优化押金支出的显示样式

## Implementation Notes

主要修改 `web/src/pages/Income.tsx` 文件中的以下部分：
1. 占比计算逻辑
2. 数据格式化函数
3. 表格渲染逻辑
