## Context

当前 ddd_domain 项目已有一个基础的 React+TypeScript 前端应用，但功能不完整。参考项目 house 系统提供了完整的租房管理功能，包括仪表盘、租约管理、账单管理、收入查询和打印功能。

**当前状态：**
- 已有基础页面架构（登录、仪表盘、位置、房间、房东、租约、账单）
- 使用 Ant Design 组件库和 TypeScript
- 与后端 API 有基础集成

**参考项目优势：**
- 完整的用户界面设计
- 丰富的交互功能
- 打印功能支持（RTF 格式）
- 完整的业务流程覆盖

## Goals / Non-Goals

**Goals:**
- 实现完整的仪表盘页面，显示关键指标
- 实现收入查询页面，支持按月查询汇总
- 实现打印功能页面，支持收据和合同打印
- 增强租约管理页面，添加续租、退租结算功能
- 增强账单管理页面，添加水电费和押金管理
- 优化整体界面设计和用户体验
- 确保所有功能与后端 API 完整集成

**Non-Goals:**
- 不重构现有的后端 API
- 不改变项目的技术栈（React+TypeScript+Ant Design）
- 不实现新的后端业务逻辑

## Decisions

### 1. 页面架构优化

**决定：** 基于现有页面架构进行增强，不重构整体布局

**理由：**
- 现有架构已经使用 Ant Design 的 Layout 组件
- 路由结构清晰，易于扩展
- 代码结构符合 React+TypeScript 最佳实践

**实现方案：**
- 在现有 `Layout.tsx` 基础上优化侧边栏导航
- 新增页面组件到 `web/src/pages/` 目录
- 保持与现有样式和布局的一致性

### 2. 打印功能实现

**决定：** 使用浏览器打印功能 + RTF 格式输出

**理由：**
- 参考项目使用 RTF 格式，可被 Word 等软件打开
- 避免引入复杂的 PDF 生成库
- 保持与后端 API 的一致性

**实现方案：**
- 使用 `<iframe>` 加载 RTF 内容
- 提供打印按钮触发浏览器打印
- 处理打印样式和分页

### 3. 数据可视化

**决定：** 使用 Ant Design 的 Statistic 和 Table 组件

**理由：**
- 现有项目已使用 Ant Design
- 无需引入额外的数据可视化库
- 保持技术栈一致性

**实现方案：**
- 使用 Statistic 组件显示仪表盘指标
- 使用 Table 组件展示列表数据
- 使用 Charts 组件（可选）显示收入趋势

### 4. 表单优化

**决定：** 使用 Ant Design 的 Form 组件 + 自定义验证

**理由：**
- 现有项目已使用 Ant Design Form
- 支持复杂的表单验证和布局
- 与现有代码风格一致

**实现方案：**
- 增强表单验证逻辑
- 优化表单布局和交互
- 添加日期选择、金额输入等组件

## Risks / Trade-offs

**风险 1：API 兼容性**
- **风险：** 后端 API 可能与参考项目不完全一致
- **缓解：** 在开发前验证 API 接口，必要时调整前端适配

**风险 2：打印功能兼容性**
- **风险：** 不同浏览器对 RTF 格式的打印支持可能不同
- **缓解：** 测试主要浏览器的打印效果，提供下载选项

**风险 3：数据加载性能**
- **风险：** 大量数据可能导致页面加载缓慢
- **缓解：** 实现分页加载和虚拟滚动

**风险 4：代码维护性**
- **风险：** 快速开发可能导致代码质量下降
- **缓解：** 遵循现有代码规范，添加必要的注释
