## Context

后端已完成架构调整，包括：
- 业务逻辑收敛到领域服务
- 添加了操作日志功能
- 添加了租约到期定时任务

但前端页面未同步更新，存在以下问题：
- 部分 API 返回格式不匹配
- 分页查询结果格式不一致
- 字段名大小写不匹配（snake_case vs camelCase）

## Goals / Non-Goals

**Goals:**
- 修复所有 CQRS handler 的返回格式
- 确保前端 API 调用与后端一致
- 修复所有前端页面的数据展示
- 逐个修复各领域模块

**Non-Goals:**
- 不修改后端业务逻辑
- 不修改数据库结构
- 不添加新功能

## Decisions

### 1. 修复策略：逐个模块修复

**决策：** 按领域模块逐个修复（landlord → location → room → lease → bill → print）

**理由：**
- 降低风险，便于验证
- 确保每个模块都能正常工作
- 便于回滚单个模块

**替代方案：**
- 一次性全部修复 - 风险高，难以验证

### 2. 数据格式：保持后端不变，调整前端

**决策：** 保持后端 API 格式不变，调整前端以匹配后端。

**理由：**
- 后端格式已经过重构，更合理
- 避免再次修改后端
- 前端改动相对独立

### 3. 字段映射：前端使用 camelCase，后端使用 snake_case

**决策：** 在前端 API 层进行字段名转换。

**理由：**
- 符合 JavaScript/TypeScript 惯例
- 符合 Go 惯例
- API 层作为适配层

## Risks / Trade-offs

**[风险] 遗漏某些字段** → 仔细对比，逐个页面测试

**[风险] API 调用失败** → 保留浏览器控制台日志，便于调试

**[风险] 分页显示异常** → 统一所有列表页的分页逻辑
